# 校园班牌系统 - 项目完整文档

## 一、项目概述

### 1.1 系统简介

校园班牌系统是一个完整的校园信息化解决方案，旨在通过现代化信息技术提升校园管理效率和信息化水平。系统采用前后端分离架构，包括后台管理系统（Web）、前端展示端（Android一体机App）两大核心部分。

### 1.2 核心目标

- **实时展示**：在教室一体机上实时展示课程安排、当前课程、天气信息、班级信息等
- **统一管理**：通过Web管理后台统一管理用户、班级、教室、课表、通知等
- **自动化运行**：Android一体机App自动启动、全屏展示、定时刷新数据
- **离线支持**：网络中断时仍能显示最近缓存数据

### 1.3 部署方式

| 部署端 | 技术栈 | 说明 |
|--------|--------|------|
| 后台管理系统 | Web (Vue.js) | 供管理员使用 |
| 前端展示端 | Android App | 一体机上运行，内嵌Web页面自动全屏展示 |

### 2.2 技术栈

#### 后端服务 (Spring Boot)
- **核心框架**: Spring Boot 3.2.0
- **ORM框架**: MyBatis Plus 3.5.5
- **数据库**: MySQL 8.0+ (开发环境使用H2内存数据库)
- **安全认证**: Spring Security + JWT
- **API文档**: SpringDoc OpenAPI 3
- **工具类**: Hutool、Lombok、EasyExcel

#### 前端管理界面 (Vue.js)
- **核心框架**: Vue 3.4
- **构建工具**: Vite 5.0
- **UI组件库**: Element Plus 2.4
- **状态管理**: Pinia 2.1
- **路由框架**: Vue Router 4.2
- **HTTP客户端**: Axios 1.6

#### Android应用 (Kotlin)
- **开发语言**: Kotlin
- **UI框架**: Jetpack Compose
- **架构模式**: MVVM + Clean Architecture
- **网络请求**: Retrofit 2.9 + OkHttp 4.12
- **图片加载**: Coil 2.5
- **数据存储**: DataStore Preferences
- **异步编程**: Kotlin Coroutines

## 三、功能模块

### 3.1 系统角色

| 角色 | 职责 |
|------|------|
| 系统管理员 | 管理用户、角色、部门、基础数据等 |
| 教师/班主任 | （未来可扩展）查看或维护本班课表、通知等 |
| 学生/访客 | 仅查看班牌展示内容（只读） |

> 当前阶段以后台管理 + 一体机展示为主，暂未涉及教师端操作。

### 3.2 模块1：班牌后台管理（Web管理平台）

#### 1.1 系统管理 → 用户管理

- **用户列表**
  - 分页展示用户信息
  - 字段：账号、用户名、所属部门、状态（启用/冻结）
  - 操作列：修改密码、初始化密码（重置为123456）、冻结/解冻、编辑、删除

- **用户查询**
  - 支持按账号、用户名、部门进行模糊搜索

- **用户新增**
  - 选择部门
  - 输入账号、用户名
  - 初始密码默认为123456
  - 创建后自动启用

- **用户修改**
  - 可修改：所属部门、账号、用户名
  - （注：密码通过"修改密码"单独操作）

- **用户删除**
  - 支持逻辑删除

- **用户角色分配**
  - 选中用户后，右侧显示树状角色列表（支持多级角色结构）
  - 可勾选一个或多个角色，实现权限分配

#### 1.2 教务管理（框架已搭建）

- 班级管理
- 教室管理
- 课表管理
- 通知公告管理

### 3.3 模块2：一体机APP（内嵌Web展示端）

#### 2.1 核心展示内容

| 展示项 | 说明 |
|--------|------|
| 当前日期 & 时间 | 实时显示，精确到秒 |
| 当日天气 | 显示城市、温度、天气图标（晴/雨/阴等）、空气质量 |
| 教室信息 | 教室编号（如：高一(3)班教室、302教室） |
| 班级信息 | 班级名称、班主任、班级口号/格言（可配置） |
| 今日课表 | 按节次列出全天课程（含课程名称、任课教师、上课时间） |
| 当前课程 | 高亮显示正在上的课（如：第3节 · 数学 · 张老师 · 10:00-10:45） |
| 下节课程 | 提前预告下一节课信息 |
| 校园通知 | 滚动公告栏，显示学校/年级发布的通知（如放假、活动等） |

#### 2.2 技术实现要求

- Android App 启动后自动全屏，禁用返回键、状态栏
- 内嵌 WebView 加载指定URL
- 支持自动刷新（如每30秒或每节课切换时更新）
- 离线缓存机制（网络中断时仍能显示最近数据）
- 自适应屏幕分辨率（适配不同尺寸一体机）

#### 2.3 配置管理（后台支持）

- 教室与班级绑定关系维护
- 课表导入/同步（支持Excel批量导入或对接教务系统API）
- 天气城市绑定（每个教室可设置所在城市）
- 通知发布功能（富文本编辑，设置有效期）

## 六、快速开始

### 6.1 后端服务启动

#### 步骤1：配置数据库（可选，开发环境使用H2）

如果使用MySQL数据库，请先创建数据库：

```sql
CREATE DATABASE classboard DEFAULT CHARACTER SET utf8mb4;
```

然后修改 `backend/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/classboard?useUnicode=true&characterEncoding=utf8
    username: root
    password: your_password
    driver-class-name: com.mysql.cj.jdbc.Driver
```

#### 步骤2：启动后端服务

```bash
cd backend
mvn spring-boot:run
```

服务启动后：
- API地址: http://localhost:8080/api
- API文档: http://localhost:8080/api/swagger-ui.html
- H2控制台: http://localhost:8080/api/h2-console (开发环境)

#### 步骤3：初始化数据

数据库会在启动时自动创建表结构。如需初始化测试数据，请执行：

```bash
mysql -u root -p classboard < backend/src/main/resources/db/init.sql
```

默认账号：**admin / 123456**

---

### 6.2 前端管理界面启动

#### 步骤1：安装依赖

```bash
cd backend/admin-web
npm install
```

#### 步骤2：启动开发服务器

```bash
npm run dev
```

访问地址: http://localhost:3000

#### 步骤3：构建生产版本

```bash
npm run build
```

构建产物在 `dist` 目录，可部署到Nginx。

### 7.1 用户管理接口

| 接口路径 | 方法 | 说明 | 参数 |
